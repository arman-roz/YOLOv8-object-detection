# YOLOv8n Convergence Study — LVIS Greedy-50

A convergence study of the **YOLOv8n object detection model** trained **from scratch** (no pretrained weights) on progressively larger subsets of the LVIS dataset. The goal is to analyze how much data the model needs to converge, and to compare results against a parallel SSD300 study trained on the same splits.

---

## Project Overview

| Property | Detail |
|---|---|
| Model | YOLOv8n (nano) |
| Backbone | CSPDarknet (randomly initialized — no pretrained weights) |
| Dataset | LVIS Greedy-50 (50 selected classes, COCO format) |
| Metric | mAP@0.50, mAP@0.50:0.95, train/val loss curves |
| Framework | Ultralytics (PyTorch) |

## Dataset

The **LVIS Greedy-50** dataset is a curated subset of LVIS v1 containing 50 frequent classes selected to maximize unique image coverage with minimal class overlap.

| Split | Train images | Val images |
|---|---|---|
| split_005 (5%) | 2,869 | 692 |
| split_025 (25%) | 14,341 | 3,561 |
| split_050 (50%) | 28,686 | 7,143 |
| split_075 (75%) | 43,016 | 10,729 |
| split_100 (100%) | 57,370 | 14,314 |

Each split is a **strict subset** of the next — split_005 images are always included in split_025, and so on. This ensures the only variable between splits is data quantity, not class coverage. All 50 classes are guaranteed to appear in every split (both train and val).

### 50 Classes
`airplane`, `banana`, `baseball_bat`, `baseball_glove`, `bed`, `bench`, `bird`, `boat`, `bowl`, `broccoli`, `bus_(vehicle)`, `cat`, `cellular_telephone`, `clock`, `computer_keyboard`, `cow`, `dog`, `doughnut`, `drawer`, `elephant`, `faucet`, `fireplug`, `fork`, `frisbee`, `giraffe`, `horse`, `kite`, `motorcycle`, `necktie`, `pizza`, `plate`, `sheep`, `skateboard`, `ski_pole`, `snowboard`, `sofa`, `spectacles`, `street_sign`, `suitcase`, `surfboard`, `teddy_bear`, `television_set`, `tennis_racket`, `toilet`, `traffic_light`, `train_(railroad_vehicle)`, `umbrella`, `vase`, `windshield_wiper`, `zebra`

## Scripts

### `convert_to_yolo.py`

**Run once before training.** Converts COCO-format annotations to the YOLO label format that Ultralytics expects, then generates a `data/<split>.yaml` config file for each split.

**What it does:**
- Reads `annotations.json` from each split's `train/` and `val/` folders
- Builds a consistent LVIS `category_id` → YOLO class index mapping (0–49), sorted by category ID so the mapping is identical across all splits
- Converts each bounding box from COCO format `[x_top_left, y_top_left, width, height]` to YOLO format `[x_center, y_center, width, height]` (all values normalized to 0–1 relative to image dimensions)
- Writes one `.txt` label file per image into a `labels/` folder alongside each `images/` folder
- Writes `data/split_005.yaml` through `data/split_100.yaml` — each pointing to the correct train and val image paths

```bash
python convert_to_yolo.py
```

---

### `train.py`

Trains YOLOv8n **from scratch** on a chosen split. Change the one `SPLIT` variable at the top to switch between splits.

**Key design decisions:**

**No pretrained weights** — the model is loaded from `yolov8n.yaml` (architecture definition only), which initializes all weights randomly. This is the critical difference from the typical YOLOv8 usage where a `.pt` file with COCO-pretrained weights is used.

**Early stopping** — training automatically stops when the monitored validation metric (fitness score: weighted combination of mAP@0.50 and mAP@0.50:0.95) does not improve for `PATIENCE=10` consecutive epochs. This lets each split find its natural convergence point rather than running for a fixed number of epochs.

**SGD optimizer** — matches the SSD300 study setup (SGD, lr=1e-3, momentum=0.9, weight_decay=5e-4) so the two models are trained under comparable conditions.

**LR warmup** — the learning rate is linearly ramped from near-zero to the target lr over the first 3 epochs, preventing unstable gradient updates when weights are random and gradients are large.

**CSV loss logging** — a custom callback writes `results/<split>/loss_log.csv` after every epoch, recording train/val box loss, classification loss, DFL loss, mAP@0.50, and current learning rate. This mirrors the SSD study's `loss_log.csv` for direct comparison.

**Periodic checkpoints** — weights are saved every 50 epochs (in addition to `best.pt` and `last.pt`), matching the SSD checkpointing interval.

```bash
# In train.py, set:
SPLIT = 'split_005'  # change to split_025, split_050, split_075, or split_100

python train.py
```

| Parameter | Value |
|---|---|
| Max epochs | 500 (safety ceiling) |
| Early stopping patience | 10 epochs |
| Image size | 640 × 640 |
| Batch size | 8 |
| Optimizer | SGD |
| Learning rate | 1e-3 |
| Momentum | 0.9 |
| Weight decay | 5e-4 |
| LR warmup | 3 epochs |
| Pretrained | No |

---

### `eval.py`

Evaluates a trained checkpoint on the validation set. Prints per-class AP@0.50 and overall mAP@0.50 / mAP@0.50:0.95.

Set `EPOCH = None` to evaluate the best checkpoint, or an integer to evaluate a specific periodic snapshot.

```bash
# In eval.py, set:
SPLIT = 'split_005'
EPOCH = None   # or e.g. 50, 100, 150 for a periodic snapshot

python eval.py
```

---

### `detect.py`

Runs object detection on a single image or directory of images using a trained checkpoint. Saves annotated output images to `results/<split>/detections/`.

```bash
# In detect.py, set:
SPLIT  = 'split_005'
SOURCE = 'path/to/image.jpg'   # or a directory

python detect.py
```

## Quickstart

```bash
# 1. Convert annotations (one-time setup)
python convert_to_yolo.py

# 2. Train — change SPLIT in train.py to switch splits
python train.py

# 3. Evaluate best checkpoint
python eval.py

# 4. Run detection on an image
python detect.py
```

## Loss Log Format

`results/<split>/loss_log.csv`:

```
epoch, train_box_loss, train_cls_loss, train_dfl_loss, val_box_loss, val_cls_loss, val_dfl_loss, mAP50, lr
1, 3.1420, 4.2310, 1.8821, 3.5210, 4.8830, 2.1023, 0.0012, 0.001000
2, 2.9810, 3.9920, 1.7430, 3.3100, 4.6210, 1.9870, 0.0025, 0.001000
...
```

**Loss components:**
- `box_loss` — bounding box regression loss (distribution focal loss applied to box coordinates)
- `cls_loss` — classification loss (binary cross-entropy per class)
- `dfl_loss` — distribution focal loss (helps predict precise box edges)

## Checkpoints

Saved to `results/<split>/weights/`:

| File | When saved | Purpose |
|---|---|---|
| `best.pt` | When val fitness improves | Best model — used for evaluation |
| `last.pt` | Every epoch | Crash recovery — always resumable |
| `epoch50.pt`, `epoch100.pt`, ... | Every 50 epochs | Convergence snapshots |

## Comparison with SSD300

This study is designed to be directly comparable to a parallel SSD300 convergence study trained on the same LVIS Greedy-50 splits:

| Property | SSD300 | YOLOv8n |
|---|---|---|
| Backbone | ResNet50 | CSPDarknet |
| Initialization | Kaiming uniform (from scratch) | Random (from scratch) |
| Optimizer | SGD | SGD |
| Learning rate | 1e-3 | 1e-3 |
| Momentum | 0.9 | 0.9 |
| Weight decay | 5e-4 | 5e-4 |
| Early stopping patience | 10 epochs | 10 epochs |
| Image size | 640 | 640 |
| Batch size | 8 | 8 |
| Loss logged | train/val total loss | train/val box + cls + dfl loss |
| Metric | mAP@0.50 | mAP@0.50 / mAP@0.50:0.95 |
